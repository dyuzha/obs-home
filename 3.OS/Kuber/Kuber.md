[[kubectl]] - CLI интерфейс к API Server

**Control Plane**
[API Server](api-server) - центральная точка управления
[Scheduler](scheduler) -  решает на какую node отправить pod
[Control Manager](control-manager) - следит за desired state
[Etcd](etcd) - key-value база (хр. состояние файла)

**Worker nodes**
[Kubelet](kubelet) - Агент контейнера на каждой ноде (общается с runtime через CRI)
[Container runtime](container-runtime) Программа, запускающая контейнеры (containerd, CRI-O)
[[CNI]] - Сеть контейнера
[Kube-proxy](kube-proxy) - Реализует абстракцию на уровне сети
[Pods](pod) - Минимальная единица развертывания


Система оркестрации контейнеров
- Декларативное управление
- Масштабируемость
Легко добавлять/удалять ресурсы (поды узлы)
- Отказоустойчивость
    - Автоматическое восстановление упавших контейнеров
    - Поддержка реплик, балансировка нагрузки
- Оркестрация и автоматизация
    Деплой, обновления, мониторинг и откаты - автоматически
- Абстрация ресурсов
Приложения не зависят от сервера (миграция внутри кластера)


## k3s vs k8s
---

| Компонент       | k8s               | k3s            |
|-----------------|-------------------|----------------|
| API Server      | Отдельный процесс | Встроен        |
| Scheduler       | Отдельный процесс | Встроен        |
| Control manager | Отдельный процесс | Встроен        |
| Etcd            | Требует установки | Встроен SQLite |

| Компонент         | k8s                           | k3s                     |
|-------------------|-------------------------------|-------------------------|
| Kubelet           | Отдельный процесс             | Встроен      |
| Container runtime | Отдельный процесс (любой CRI) | Встроен containerd      |
| CNI           | Любой плагин                  | Интегрированный Flannel |
| kubectl           | Отдельно устанавливается      | Встроен как k3s kubectl |


**K3s**:
- Отдельный бинарник вместо десятка процессов
- Встроен traefik как ingress (можно отключить)
- Установка через скрипт
- Автоматически управляет TLS и kubeconfig

**K8s**:
- Отдельные сервисы
- Требует etcd как отдельный сервис
- CNI и container-runtime настраиваются вручную
- kubectl ставится отдельно
