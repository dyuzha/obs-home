# Модель OSI
---
L1-L3 - уровень среды
Сосредоточены на передачи данных через различные среды: кабеля, сети, ...
Обеспечивают физическую и логическую связь между устройствами.

L4-L7 - уровень хоста
Функционируют на конечных устройствах.
Обрабатываю данные и обеспечивают работу приложений.

## Сеансовый (Данные) Session Layer
---
Управление сенасом связи между приложениями. Обеспечивает установку, поддержание и завершение сеансов, а также синхронизацию и управление обменом данными.

## Транспортный (сегменты, датаграммы) Transport Layer
---
Обеспечивает надежную передачу данных.
Выполяет сегментацию и повторную сборку данных.
Гарантирует что данные передаются без потерь и дупликатов.

- Сборка

UDP
SCTP

## Сетевой (пакеты) Network Layer
---
Отвечает за маршрутизацию пакетов данных между узлами сети.
Обеспечивает логическую адресацию и определяет пути передачи данных через различные сети.
- Определение пути и ip (логическая адресация)
- Фрагментация

**IP**
**ARP**  - протокл, используемый для сопоставления IP-адресов с физическими MAC-адресами устройств в LAN.
**IPsec** - Наобор протокол для юезеопасной передачи данных по IP сетям.
- Аутентификация;
- Целостность данных;
- Шифрование.

**ICMP**  (Internet Control Message Protocol) - протокол сообщений управления в интернете.
**IGMP**  (Interner Group Managment Protocol) - Протокол управления группами интернета.
- Управление членствами в мультикаст-группы.
*Мультикаст это метод передачи данных от одного отправителя к нескольким получателям одновременно.*
- Позволяет маршрутизаторам узнавать, какие устройства в локальной сети хотят получать данные от конерретных мультикаст групп

**IGP**  (Interior Gateway Protocol)
**OSPF** (Протокол открытого кратчайшего пути первым)
- Протокол маршрутизации, используемый для определения наиболее эффективного пути передачи данных через IP-сеть.
- Алгоритм Дейкстры.

[[bgp]] (Border Gateway Protocol) -  Протокол маршрутизации между автономными системами в Интернете.

**RIP** (Routing Information Protocol)
Протокол маршрутизации, использующий алгоритм расстояинк-вектор. RIP обновляет маршрутизационные таблицы через определенные интервалы времени, что делает его подходящим для небольших сетей.

**MTU** - Maximum Transmission Unit
**NAT** - позволяет нескольким устройствам в локальной сети использовать один общий IP
**DHCP** - Протокол динамической конфигурации устройств.
**SDN** - Позволяет централизованно управлять сетевой инфраструктурой с помощью ПО

### Пакет
---
Header:
- версия IP
- Длина заголовка
- Type of Service - приоритет.
- Total Length - общая длина.
- Identification - уникальный идентификатор пакета.
- Flags - для управления фрагментацией.
- Fragment Offset - смещение фрагмента (положение фрагмента в исходном пакете).
- Time to Live - время жизни пакета (макс число прыжков)
- Protocol -  протокол транспортного уровня (TCP / UDP)
- Header-checksum
- src-IP
- dst-IP
- Options


Data:
- payload (сегменты, датаграммы, ...)


## Канальный (кадры/фреймы) Data Link Layer
---
Обеспечивает:надежную передачу данных через физический канал.
- Обнаружение и исправлению ошибок
- Доступ к среде передачи

Передача данных в рамках одной локальной подсети

**MAC**  (Mediuem Access Control) - подуровень L2
Метод контроля доступом к физической среде.
Например: CSMA/CD

**LLC**  (Logical Link Control) - подуровень L2
Отчечает за управление логическими соединениями, кадрами, контроль ошибок.
Это интерфейс между сетью и MAC-подуровнем



**Ehternet**
**коммутаторы**
**802.11**  (WLAN) - стандарт беспроводных локальных сетей
**VLAN**  - технология, позволяющая разделять одну физическую сеть на несколько виртуальных сетей.
**FrameReplay**  - технология для межсетевого взаимодействия. Передает данные в виде кадров через виртупльные сети.

**PPP**  - технология для установки прямого соединения между 2 узлами:
- Аутентификация
- Шифрование
- Компрессия

**Token Ring** - технология. Устройства подключены в кольцо. Передача данных осуществляется по принципу передачи token'а. Только устройство, владеющие токеном, может передавать данные


Прочее:
**ATN** - сеть, предназначенная ля поддержки воздушного трафика и управления воздушным движением.
**HDP** - профиль Bluetooth, для мед приборов.
**Q.921**

## Физический (Биты)
---
Определяет среду передачи данных
- Электрические импульсы
- Свет
- Радио-волны

### Типы топологии
---
1. Шина (Bus)
Все устройства подключены к одной физической среде.

2. Зекзда
Все устройства подключены к одному физическому устройству.

3. Кольцо
Устройства соедененны в кольцо, где каждый узел имеет 2 соседа.

4. Ячеистая/Сетчатая (Mesh)
a) Полносвязная - все устройства напрямую соедененны друг с другом.
b) Частатчная - некоторый устройства соединены не напрямую.

5. Древовидная топология
Комбинация звездной и шинной топологии. Устройства соединены в группы, которые соедененны центральными узлами.
