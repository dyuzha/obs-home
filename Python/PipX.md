# PipX
---

**Pipx** — это инструмент, который устанавливает и запускает приложения на Python, написанные в виде пакетов, **в изолированных средах**. 
Ключевая идея: "установить приложение глобально, но безопасно".

Проще говоря:

- `pip install some-tool`: 
	- Устанавливает пакет и все его зависимости прямо в ваше основное окружение Python (системное или пользовательское). 
	*Это может привести к конфликтам версий и "захламлению" окружения.*

- `pipx install some-tool`: 
	- Создает для отдельную виртуальную среду, 
	- Устанавливает пакет (инструмент) туда, 
	- Делает его команду доступной глобально в вашей системе (через ссылки в `~/.local/bin`).

## Ключевые особенности и преимущества
---

1. **Изоляция (Главное преимущество):** 
Каждое приложение живет в своей собственной виртуальной среде. Зависимости разных приложений не конфликтуют между собой.

2. **Безопасность:** 
Вы не изменяете ваше основное/системное окружение Python, что снижает риск его повредить.

3. **Глобальная доступность:** 
После установки через `pipx` командой приложения можно пользоваться из любого места в терминале, как и любыми другими системными утилитами.

4. **Управляемость:** 
Легко устанавливать, обновлять, удалять и просматривать список установленных инструментов.

## Для чего идеально подходит PipX?
---

Он создан для установки **консольных утилит и приложений**, которыми вы пользуетесь как самостоятельными программами. Классические примеры:

- **Средства разработки:** 
	- `black` (форматтер), 
	- `isort` (сортировка импортов), 
	- `flake8` (линтер), 
	- `mypy` (проверка типов), 
	- `pytest` (можно установить глобально для быстрого запуска в проектах).

- **Утилиты для работы:** 
	- `httpie` (современный cURL), 
	- `cookiecutter` (генератор проектов из шаблонов), 
	- `youtube-dl`, 
	- `ansible-core`, 
	- `tox`, 
	- `pipenv`.

- **Инструменты из экосистемы Python:** 
	- `jupyter`, 
	- `jupyterlab`, 
	- `streamlit`.

- **Любые пакеты, которые предоставляют "точки входа" (entry points):** 
	- консольные команды.
